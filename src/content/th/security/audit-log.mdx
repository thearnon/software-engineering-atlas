## ใช้จริงในระบบองค์กรยังไง

Audit Log คือบันทึกเหตุการณ์สำคัญในระบบ เช่น ใครสร้างคำขอ ใครอนุมัติ ใครแก้สิทธิ์ และเกิดขึ้นเมื่อไร

ใน internal tool ขององค์กร audit log ช่วยให้ตรวจสอบย้อนหลังได้เมื่อเกิดข้อผิดพลาด ข้อโต้แย้ง หรือ security review

## เชื่อมกับ RBAC ยังไง

RBAC บอกว่าใครทำ action ได้ ส่วน audit log บอกว่า action นั้นเกิดขึ้นจริงโดยใครและในบริบทไหน

ถ้ามี permission แต่ไม่มี audit log ระบบจะรู้แค่ว่าใครควรทำได้ แต่ไม่รู้ว่าใครทำอะไรไปแล้ว

## สิ่งที่ควรระวัง

- อย่า log เฉพาะ action สำเร็จ ควรเก็บ action สำคัญที่ถูกปฏิเสธด้วย
- อย่าเก็บข้อมูลลับเกินจำเป็นใน log
- ต้องมี timestamp, actor, action, target และ context ที่พอสอบกลับได้
- Audit log ควร query ได้เมื่อ support, compliance หรือ incident review ต้องใช้
